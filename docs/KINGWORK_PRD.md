# KingWork 技能设计 PRD

## 一、项目概述

### 1.1 项目背景
售前工作涉及大量的客户沟通、项目跟进、学习成长等事务，需要高效记录和管理。KingWork 技能旨在通过智能化的工作记录和管理系统，提升售前工作效率，实现"用户最小化输入，系统最大化整理产出，积累时间复利"的目标。

### 1.2 核心目标
- **用户最小化输入**：支持自然语言输入，自动完成分类、字段填充、数据分发
- **系统最大化整理产出**：自动分析聊天记录、会议记录，智能提取关键信息
- **积累时间复利**：自动生成日报周报月报，提取有价值的洞察和惊喜内容

## 二、用户角色和使用场景

### 2.1 目标用户
- **主要用户**：售前工程师、客户经理、销售顾问
- **使用场景**：日常工作中记录客户沟通、项目进展、学习笔记、待办事项等

### 2.2 核心使用场景

#### 场景1：快速记录工作日记
**用户输入**：
```
今天下午和某某公司王总电话沟通了新项目的需求，他们希望下周安排演示，我需要准备一下演示材料
```

**系统自动处理**：
1. 识别工作类型：客户跟进
2. 提取关键信息：
   - 客户名称：某某公司
   - 联系人：王总
   - 沟通方式：电话
   - 关键内容：新项目需求、下周演示
3. 自动创建记录：
   - 写入【日记记录】表（通过 WPS365 Skill）
   - 写入【客户跟进记录】表
   - 创建待办：准备演示材料
   - 更新【客户档案】表���最近跟进时间

#### 场景2：AI 自动分析当天工作
**触发方式**：手动触发或定时任务

**系统自动处理**：
1. 获取当天的聊天记录（调用 WPS365 Skill: im search-messages）
2. 获取当天的会议记录（调用 WPS365 Skill: meeting list）
3. 调用大模型分析每条消息/会议：
   - 是否是工作相关内容
   - 工作类型分类
   - 提取关键信息（客户、项目、待办等）
4. 与已有记录对比，去除重复内容（相似度>70%）
5. 自动创建新记录到对应的数据表
6. 分析沟通协作模式：
   - 和哪些人沟通最多
   - 经常讨论的关键词
   - 写入【惊喜沟通记录】表
7. 获取当天访问的文档：
   - 提取文档主题
   - 写入【惊喜文档记录】表

#### 场景3：智能提醒
**触发方式**：手动触发或定时任务

**系统输出**：

2. [中优先级] 提交周报
   - 到期时间：2026-03-19 18:00
   - 提示：已逾期1天，请立即处理

### 👥 客户跟进提醒（2家）

1. **某某公司** - 未跟进18天
   - 最近跟进：2026-03-02

**用户交互**：
- 输入"第1条完成了" → 自动标记待办为已完成

## 四、大模型提示词工程

### 4.1 提示词设计原则

1. **结构化**：使用清晰的 JSON 输出格式，便于程序解析
2. **可验证**：输出包含置信度，便于判断结果可靠性
3. **可解释**：输出包含判断原因，便于调试
4. **少样本**：在提示词中提供示例，提高准确率
5. **上下文**：提供足够的上下文信息（时间、发送者等）

### 4.2 提示词模板管理

创建 `config/prompts.yaml` 文件，集中管理所有提示词：

```yaml
# config/prompts.yaml

prompts:
  work_classification: |
    你是一个工作日记智能分析助手...

  content_analysis: |
    你是一个工作内容智能分析助手...

  similarity_check: |
    你是一个文本相似度分析助手...

  communication_analysis: |
    你是一个沟通价值分析助手...

  report_generation: |
    你是一个工作报告生成助手...
```

### 4.3 提示词版本管理

- 每个提示词版本记录在 `config/prompts/` 目录下

def analyze_with_llm(prompt: str) -> dict:
    """
    通过 OpenClaw 调用大模型进行分析
    """
    # 获取 OpenClaw 的内置大模型调用接口
    response = openclaw.chat_completion(

## 十、风险和挑战

### 10.1 大模型准确性
**风险**：分类不准确、提取错误
**应对**：
- 提供用户纠正机制
- 收集反馈持续优化提示词
- 设置置信度阈值，低置信度时询问用户

### 10.2 性能问题
**风险**：大模型调用耗时、成本高
**应对**：
- 批量处理减少调用次数
- 缓存常用分析结果

### 10.3 数据隐私
**风险**：工作内容可能包含敏感信息
**应对**：
- 数据存储在用户企业账号下的 WPS 多维表
- 提供数据删除功能
